# Builder objective redesign — frontend (onda 0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `max_return_cvar` (maximize return subject to a user CVaR ceiling) the Builder's default objective, driven by a mandate selector that pre-fills an editable daily CVaR-ceiling input, and surface the effective (regime-adjusted) ceiling in the results.

**Architecture:** Three source files change (`assets.ts`, `BuilderView.tsx`, `ResultsPanel.tsx`) plus a regenerated `api.d.ts`. The request-body fields the UI needs (`objective: "max_return_cvar"`, `mandate`, `cvar_limit`) ALREADY exist in the generated contract; the regeneration only adds `cvar_limit_effective` and `regime_state` to `DiagnosticsOut`. The CVaR preset ladder per mandate lives entirely in the frontend as UI defaults — the backend receives the final `cvar_limit`.

**Tech Stack:** Next.js 15 / React 19, TypeScript (strict, `tsc --noEmit`), `@tanstack/react-query`, Tailwind v4 + Carbon, vitest 4 (+ jsdom, Testing Library), openapi-typescript 7.

**Test command:** from repo root, one suite: `cd frontend && pnpm exec vitest run src/components/builder/<file>.test.ts` (or `.test.tsx`). The full suite is `cd frontend && pnpm test` (`vitest run`). React component suites declare `// @vitest-environment jsdom` as their first line; pure-logic suites run in the default `node` env.

**Frontend gate command:** `cd frontend && pnpm lint && pnpm run typecheck` (scripts verified in `frontend/package.json`: `"lint": "eslint"`, `"typecheck": "tsc --noEmit"`), then the test command above.

**Spec:** `docs/superpowers/specs/2026-06-17-builder-objective-redesign-design.md`
**Backend plan (dependency):** `docs/superpowers/plans/2026-06-17-builder-objective-redesign-backend.md`
